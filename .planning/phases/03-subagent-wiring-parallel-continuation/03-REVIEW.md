**Reviewed:** 2026-04-27T20:00:00Z

## Summary

Phase 3 adds `fork()` / `_ForkCM` to `TrajectoryRecorder` for parallel task groups (sibling trajectory files), `_safe_descriptor` for slug sanitization, `create_dispatch_step` for linking siblings via `SubagentTrajectoryRef`, and wires these into three fan-out call sites (`phase_fix_parallel`, `phase_per_stack_reviews`, `pre_scan`). The ContextVar isolation is correct -- `_RECORDER_VAR.set(child)` inside `_ForkCM.__aenter__` and `_RECORDER_VAR.reset(token)` in the finally of `__aexit__` properly scope the child recorder to the fork block, and asyncio's per-task ContextVar copy semantics ensure isolation across `tg.start_soon` tasks.

The main concerns are: (1) `_safe_descriptor` returns empty string for degenerate inputs, producing malformed filenames; (2) significant code duplication across all three wiring sites; and (3) `create_dispatch_step` has an uncaught `ValueError` path from `Path.relative_to`.

## Critical Issues

### CR-01: `_safe_descriptor` returns empty string for degenerate inputs, producing malformed filenames

**File:** `daydream/trajectory.py:53-57`
**Issue:** `_safe_descriptor("")` returns `""`, and `_safe_descriptor("...")` also returns `""` (all characters stripped). This flows through `_sibling_path_for` at line 464 to produce a filename like `abcdef12..json` (double-dot, no descriptor segment). While current call sites use hardcoded non-empty descriptors like `"fix-0"` and `"deep-python"`, this is a latent bug: any future caller passing an empty or all-punctuation descriptor will silently produce a malformed file path. More critically, if the function is ever reused for user-supplied input, the empty slug creates a `..json` filename that could confuse path-based tooling.
**Fix:**
```python
def _safe_descriptor(raw: str) -> str:
    """Slugify a descriptor to filesystem-safe characters (D-06)."""
    slug = re.sub(r"[^a-z0-9-]", "-", raw.lower())
    slug = re.sub(r"-{2,}", "-", slug)
    slug = slug.strip("-")
    if not slug:
        raise ValueError(f"Descriptor {raw!r} produces empty slug after sanitization")
    return slug
```

## Warnings

### WR-01: `create_dispatch_step` has uncaught `ValueError` from `Path.relative_to`

**File:** `daydream/trajectory.py:490`
**Issue:** `sibling_path.relative_to(self.target_dir / ".daydream")` raises `ValueError` if `sibling_path` is not under the expected base directory. While `_register_sibling` on line 584 only registers when `child.path.exists()`, and paths are normally generated by `_sibling_path_for` (which always produces paths under `.daydream/trajectories/`), the `child.path` attribute is publicly mutable (test `test_fork_write_failure_degrades` demonstrates this at line 752 by setting `child.path = Path("/nonexistent-dir-xyz/child.json")`). If a child's path is mutated and the file happens to exist at the mutated location, `create_dispatch_step` will raise an unhandled `ValueError` that propagates up through the run.
**Fix:**
```python
for sibling_path, desc in self._registered_siblings:
    try:
        rel = str(sibling_path.relative_to(self.target_dir / ".daydream"))
    except ValueError:
        rel = sibling_path.name  # fallback to filename only
    results.append(
        ObservationResult(
            content=f"Dispatched to {desc}",
            subagent_trajectory_ref=[
                SubagentTrajectoryRef(session_id=self.session_id, trajectory_path=rel),
            ],
        )
    )
```

### WR-02: Substantial code duplication across recorder/no-recorder branches at all three wiring sites

**File:** `daydream/phases.py:1013-1037`, `daydream/phases.py:1487-1523`, `daydream/exploration_runner.py:246-266`
**Issue:** Each of the three fan-out sites (`phase_fix_parallel`, `phase_per_stack_reviews`, `_run_specialist`) duplicates the entire try/except/run_agent body in both the `if recorder is not None` and `else` branches. In `phase_fix_parallel`, this is 24 duplicated lines. In `phase_per_stack_reviews`, it's 38 duplicated lines. In `_run_specialist`, it's 18 duplicated lines. This duplication means any future change to the run_agent call pattern (e.g., adding new kwargs, changing error handling) must be applied in two places at each site -- six total locations. The fork context manager is a no-op-safe construct that could be refactored to eliminate the branching entirely (e.g., a helper that yields a nullcontext when recorder is None).
**Fix:** Extract a helper that returns either a fork CM or a nullcontext:
```python
from contextlib import asynccontextmanager, nullcontext

def _maybe_fork(recorder: TrajectoryRecorder | None, descriptor: str):
    if recorder is not None:
        return recorder.fork(descriptor)
    return nullcontext()
```
Then each call site reduces to a single branch:
```python
async with _maybe_fork(recorder, f"fix-{task_index}"):
    try:
        async with limiter:
            await run_agent(...)
        ...
    except Exception as e:
        ...
```

### WR-03: `_ForkCM.__aexit__` registers sibling AFTER `finally` block, creating coupling between write success and path existence

**File:** `daydream/trajectory.py:584`
**Issue:** Line 584 (`if child.parent is not None and child.path.exists()`) runs after the `finally` block. If `_write()` partially writes the file (e.g., creates the file but fails mid-write due to disk full), `child.path.exists()` returns True and a corrupt sibling file is registered with the parent. The `_write()` method calls `path.write_text()` which is not guaranteed to be atomic -- if the process is interrupted between file creation and content flush, a zero-byte or partial file can exist. The `create_dispatch_step` would then reference a sibling trajectory that fails to parse.
**Fix:** Check file content validity rather than mere existence, or capture `_write()` success as a boolean:
```python
async def __aexit__(self, exc_type, exc_val, exc_tb):
    child = self._child
    if child is None:
        return
    write_ok = False
    try:
        child._write()
        write_ok = bool(child.steps)  # _write is a no-op when no steps
    except Exception as exc:
        print_warning(_console, f"Sibling trajectory write failed: {type(exc).__name__}: {exc}")
    finally:
        if child._previous_token is not None:
            _RECORDER_VAR.reset(child._previous_token)
            child._previous_token = None
    if write_ok and child.parent is not None:
        child.parent._register_sibling(child.path, self._descriptor)
```

## Info

### IN-01: No integration tests for fork wiring in phases.py or exploration_runner.py

**File:** `tests/test_trajectory.py`
**Issue:** The test file thoroughly covers `TrajectoryRecorder.fork()`, `_ForkCM`, `_safe_descriptor`, and `create_dispatch_step` in isolation. However, there are zero integration tests verifying the actual wiring in the three production call sites (`phase_fix_parallel`, `phase_per_stack_reviews`, `pre_scan`). The existing tests for those functions (`test_deep_fanout.py`, `test_exploration_runner.py`) do not exercise the fork paths at all -- they predate the Phase 3 changes. A regression in the wiring (e.g., `get_current_recorder()` returning None due to test isolation, or the dispatch step being placed at the wrong scope) would not be caught.
**Fix:** Add at least one integration test per wiring site that uses a real `TrajectoryRecorder` context, calls the phase function with a mock backend, and asserts that sibling files are produced and the dispatch step is present in the parent trajectory.

### IN-02: Test `test_safe_descriptor_slugification` does not cover empty-string input

**File:** `tests/test_trajectory.py:675-683`
**Issue:** The test covers various slugification patterns including path traversal (`"../etc/passwd"`), but does not test the empty-string case (`_safe_descriptor("")`), the all-dots case (`_safe_descriptor("...")`), or the all-spaces case (`_safe_descriptor("   ")`). These all produce empty strings today, which flows through to malformed filenames (see CR-01).
**Fix:** Add test cases:
```python
assert _safe_descriptor("") == ""  # documents current behavior
assert _safe_descriptor("...") == ""
assert _safe_descriptor("   ") == ""
```
Or, after CR-01 is fixed, assert they raise `ValueError`.
