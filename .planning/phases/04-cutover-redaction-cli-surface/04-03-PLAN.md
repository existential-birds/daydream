       ```python
       def _signal_handler(signum: int, frame: object) -> None:
           """Handle termination signals: flush partial trajectory then request shutdown.

           D-07: SIGINT/SIGTERM flushes a `<path>.partial` trajectory with
           ``extra.partial=true`` so consumers know the run was interrupted.
           Reads the recorder via the ContextVar (signal handlers are not in
           the async context, but ContextVar.get() works synchronously from
           any thread/context).
           """
           signal_name = signal.Signals(signum).name
           set_shutdown_requested(True)

           # Flush partial trajectory before tearing down (D-07).
           # write_partial is synchronous and exception-safe — it cannot crash
           # the shutdown path even if the disk is full or path is unwritable.
           recorder = get_current_recorder()
           if recorder is not None:
               recorder.write_partial()

           panel = ShutdownPanel(console)
           set_shutdown_panel(panel)
           panel.start(f"Received {signal_name}, shutting down")

           if get_current_backends():
               panel.add_step("Terminating running agent(s)...")

           raise KeyboardInterrupt
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User CLI input → argparse | argparse rejects unknown flags safely (existing stdlib behavior); no new attack surface |
| Signal handler → trajectory file | Synchronous file write inside SIGINT path runs with whatever filesystem permissions the user has at run start |
| ContextVar.get() → signal handler | Reading the recorder from ContextVar in a signal handler is documented-safe in the Python signal docs (signal handlers run on the main thread; ContextVar reads are atomic) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Denial of Service | SIGINT race writes partial file then __aexit__ overwrites with stale main file | mitigate | `__aexit__` always writes to `self.path`, not `self.path.partial` — separate filenames eliminate the clobber race. Phase 2 D-13 already redacts at flush time so the partial file is also redacted |
| T-04-10 | Tampering | SIGINT handler called before recorder is initialized (e.g., during arg parsing) | mitigate | `get_current_recorder()` returns `None` when no `async with TrajectoryRecorder(...)` is on the stack; `if recorder is not None:` guard skips `write_partial()` cleanly |
| T-04-11 | Information Disclosure | Promoted [PHASE2_ERROR] message text echoes parse exception (may include user file content) | mitigate | The `str(exc)` in `print_error(console, "Phase 2 Error", str(exc))` is bound to the user terminal only — does not enter trajectory file (trajectory has its own redaction in Plan 01). Acceptable per D-08 |
| T-04-12 | Loss of Information | User runs `daydream --debug` and silently gets no error | mitigate | argparse hard-reject (D-05) means `daydream --debug` exits code 2 with `unrecognized arguments: --debug`. Test `test_debug_flag_rejected` (Task 1 acceptance criterion) proves this |
